819  * @defgroup LLVMCTLTO_CACHING ThinLTO Cache Control
820  * @ingroup LLVMCTLTO
821  *
822  * These entry points control the ThinLTO cache. The cache is intended to
823  * support incremental builds, and thus needs to be persistent across builds.
824  * The client enables the cache by supplying a path to an existing directory.
825  * The code generator will use this to store objects files that may be reused
826  * during a subsequent build.
827  * To avoid filling the disk space, a few knobs are provided:
828  * - The pruning interval limits the frequency at which the garbage collector
829  * will try to scan the cache directory to prune expired entries.
830  * Setting to a negative number disables the pruning.
831  * - The pruning expiration time indicates to the garbage collector how old an
832  * entry needs to be to be removed.
833  * - Finally, the garbage collector can be instructed to prune the cache until
834  * the occupied space goes below a threshold.
835  * @{
836  */
